## Executive Summary

A Linux security alert was triggered on **LinHRWebServer03** after script execution was detected from temporary and non-standard directories.

The affected host was:

`LinHRWebServer03 / 10.0.14.12`

The activity was performed by:

`Arun Jain`

The user role was identified as:

`Linux Admin`

The activity originated from source address:

`10.1.0.12`

Splunk logs confirmed two successful script executions:

`/tmp/.patch/runme.sh`

`/dev/shm/scan.py`

Execution from `/tmp`, hidden directories, and `/dev/shm` is suspicious because attackers commonly use these locations for temporary payload staging, malware execution, scanning scripts, defense evasion, or post-exploitation activity. However, the activity was performed by a Linux Admin, and no additional malicious activity was observed in the provided logs.

Final assessment: **Suspicious script execution reviewed. No confirmed malicious activity was observed from the provided evidence, but validation with Arun Jain or the Linux Team is recommended to confirm the purpose and contents of the executed scripts.**
