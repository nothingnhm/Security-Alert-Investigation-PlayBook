## Executive Summary

A Linux security alert was triggered on **LinQAWebServer07** after successful modification and deletion activity was observed against security-relevant log files.

The affected server was:

`LinQAWebServer07 / 10.0.15.16`

The activity was performed by user:

`William Malhotra`

The user role was identified as:

`Linux Admin`

The activity originated from source address:

`10.1.1.54`

Splunk logs confirmed the following successful actions:

`vi /var/log/audit/audit.log`

`rm /var/log/auth.log`

The first action indicates that the Linux audit log was opened or edited using `vi`. The second action indicates that the authentication log file `/var/log/auth.log` was deleted using `rm`.

Modification or deletion of log files is suspicious because it can reduce investigation visibility and may indicate an attempt to hide activity, remove evidence, or tamper with audit trails. However, because the activity was performed by a Linux Admin, it may also be legitimate if performed during approved maintenance, troubleshooting, log cleanup, or log rotation.

Final assessment: **Suspicious log modification/deletion activity. Linux Team and user validation are required to determine whether the activity was authorized or possible defense evasion.**

## Analyst Assessment

**Assessment:** **Suspicious Log Modification / Deletion Activity — Linux Team Validation Required**

The activity is suspicious because security-related log files were modified and deleted successfully.

The evidence supporting the alert includes:

1. `vi /var/log/audit/audit.log` was executed successfully.
2. `rm /var/log/auth.log` was executed successfully.
3. The actions occurred within one minute.
4. Both files are important for authentication and audit investigation.

6. The commands were executed from `/home/william`.
7. The activity originated from source address `10.1.1.54`.

However, the user is identified as a **Linux Admin**, so the activity may be authorized if it was part of maintenance, troubleshooting, log cleanup, log rotation, or an approved change.

Current assessment: **Suspicious until validated. No confirmed compromise from the provided logs alone.**
